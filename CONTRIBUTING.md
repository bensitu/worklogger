# Contributing to WorkLogger

## Getting Started

1. Fork the repository.
2. Create a feature branch from your fork.
3. Install dependencies with `pip install -r requirements.txt`.
4. Run the app locally with `python -m worklogger.main`.

## Development Guidelines

- Keep changes focused and easy to review.
- Prefer English for code comments, commit messages, and documentation.
- Preserve existing user data paths and backward compatibility when possible.
- Add or update tests when you introduce behavior changes.
- Avoid committing generated files from `build/`, `dist/`, or local databases.

## Pull Requests

Please include:

- A short description of the problem and the solution
- Screenshots for UI changes when helpful
- Notes about manual verification or automated checks you ran
- Any migration or compatibility concerns

## Reporting Bugs

When possible, include:

- Operating system and version
- App version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or logs if relevant
